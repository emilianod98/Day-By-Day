# 📅 Cronograma Intensivo de HTML (4 Días)

## Día 1: Los Cimientos y el Esqueleto Semántico
El objetivo de este día es entender cómo se estructura una página web desde la raíz y dejar de usar etiquetas genéricas (como `<div>`) para todo.

- **Estructura básica:** Aprende la anatomía de un documento HTML (`<!DOCTYPE html>`,` <html>`, `<head>`, `<body>`).
- **Metadatos esenciales:** Configuración del juego de caracteres (`charset="UTF-8"`), título de la página (`<title>`) y la etiqueta viewport para diseño responsivo.
- **Etiquetas de texto y jerarquía:** Títulos (`<h1>` a `<h6>`), párrafos (`<p>`), saltos de línea (`<br>`) y énfasis (`<strong>`, `<em>`).
HTML5 Semántico: Estructura tu web con significado real usando `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>` y `<footer>`.

🚀 Reto del día: Crea la estructura semántica de un artículo de noticias o la biografía de un personaje, usando únicamente HTML5 semántico limpio. Sube el archivo .html a tu repositorio.

## Día 2: Hipervínculos, Multimedia y Tablas
Hoy aprenderás a conectar páginas entre sí y a enriquecer tu sitio web con contenido visual y datos estructurados.

- **Enlaces** (Hipervínculos): Uso de la etiqueta `<a>`, atributos href, target="_blank" para abrir en nuevas pestañas y anclas internas para navegar en la misma página.
- **Imágenes y accesibilidad:** Inserción de imágenes con `<img>`, optimización con los atributos alt (fundamental para SEO y accesibilidad), width y height.
- **Contenido multimedia:** Etiquetas nativas de reproducción de audio (`<audio>`) y video (`<video>`) con sus respectivos controles.
- **Tablas estructuradas:** Creación de tablas de datos utilizando `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>` y los atributos de fusión de celdas colspan y rowspan.

🚀 Reto del día: Diseña una página de catálogo de productos o galería multimedia, donde incluyas una tabla comparativa de precios y especificaciones. Haz tu commit diario.

## Día 3: Formularios Avanzados y Captura de Datos
Los formularios son el puente de interacción entre el usuario y tu aplicación. Hoy dominarás cómo estructurar la captura de información de manera profesional.

- **Estructura del formulario:** Uso de `<form>`, atributos action y method. Agrupación de elementos con `<fieldset>` y `<legend>`.
- **Elementos de entrada (Inputs):** Tipos de input esenciales (text, password, email, number, date, checkbox, radio, file, hidden).
- **Validación nativa:** Atributos para validar datos sin JavaScript (required, min, max, pattern, placeholder).
- **Elementos de selección y texto largo:** Menús desplegables (`<select>`, `<option>`, `<optgroup>`), listas de sugerencias (`<datalist>`) y áreas de texto extensas (`<textarea>`).

🚀 Reto del día: Crea un formulario de registro de usuarios ultra detallado (como el de una aerolínea o una aplicación bancaria) aplicando validaciones nativas estrictas. ¡Otro cuadrito verde para GitHub! 

## Día 4: HTML Avanzado, SEO Técnico y Buenas Prácticas
El último día está dedicado a optimizar tu código para los motores de búsqueda (Google) y conocer las herramientas que usan los profesionales.

- **SEO Técnico (Metatags):** Configuración de meta descripciones, etiquetas Open Graph (para que tu web se vea bien al compartir el link en redes como WhatsApp o X) y Twitter Cards.
- **Favicons y recursos externos:** Vinculación de iconos de pestañas (`<link rel="icon">`) y carga de scripts externos de forma óptima (`async` y `defer`).
- **Accesibilidad (ARIA Web):** Introducción a los atributos role y aria-* para que lectores de pantalla de personas con discapacidad visual puedan interpretar tu sitio correctamente.
- **Validación de código:** Uso del validador oficial del W3C para asegurar que tu código esté libre de errores de sintaxis.

🚀 Reto del día: Optimiza todos los archivos HTML que creaste los días anteriores, agrégales las etiquetas meta de SEO, valida que no tengan errores y escribe un archivo README.md profesional en tu repositorio explicando todo lo que aprendiste en este sprint intensivo.
