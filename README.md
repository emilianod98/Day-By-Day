# Day by day.
*Taking one step at a time.*

Oficialmente empiezo mi viaje en la programación. 🚀 Mi enfoque es claro: un commit diario, por más mínimo que sea el cambio. La constancia es el secreto para dominar cualquier habilidad. 💻


### 📅 Plan de 60 Días: Un "Commit" al Día
**Semana 1:** El Tablero y los Cimientos (Días 1 a 7)
El objetivo de esta semana es dominar la herramienta (Git/GitHub) y asentar las bases sin frustrarte.

- [green] Día 1: Crea tu cuenta de GitHub, instala Git en tu computadora, configura tu perfil y crea tu primer repositorio llamado aprendiendo-programacion. Sube un archivo README.md presentándote.

- [✔️] Día 2: Aprende los comandos básicos (git status, git add, git commit, git push). Modifica el archivo de presentación y súbelo.

- [ ] Días 3 al 7: Dedícalos exclusivamente a HTML básico. Cada día crea un archivo .html diferente probando etiquetas: un día solo textos, otro día formularios, otro día tablas, otro día imágenes y videos. Cada día subes esa práctica al repositorio.


<br> **Semana 2:** Dándole Estilo al Caos (Días 8 a 14)

- [ ] Días 8 al 12: Introducción a CSS. Cada día experimentas con propiedades distintas: colores y fuentes, el modelo de caja (margins y paddings), posicionamiento, y selectores. Un archivo .css nuevo subido cada día.

- [ ] Días 13 y 14: Flexbox y Grid (las herramientas para ordenar elementos). Diseña un menú de navegación el sábado y una galería de tarjetas el domingo. Súbelos.

<br> **Semanas 3 y 4:** Tu Primer Proyecto Real (Días 15 a 28)
Aquí vas a crear un repositorio nuevo para un proyecto que te tome dos semanas, pero subiendo los avances día por día.

***Proyecto: Tu Portfolio Web Personal.***

- [ ] Días 15 al 18 (Maquetación): Estructura en HTML la cabecera, la sección "sobre mí", la sección de proyectos y el formulario de contacto. Un commit por sección.

- [ ] Días 19 al 24 (Diseño): Aplica CSS paso a paso. Un día para los colores y tipografías generales, otro día para hacer el menú responsivo, otro día para diseñar las tarjetas de tus futuros proyectos.

- [ ] Días 25 al 28 (Detalles y Deploy): Corrige errores visuales, asegúrate de que se vea bien en celulares y usa GitHub Pages (es gratis) para que tu web esté viva en internet. Cada ajuste es un commit.

<br> **Semanas 5 y 6:** El Cerebro del Programador - JavaScript (Días 29 a 42)
Volvemos a las prácticas diarias de lógica. Crea un repositorio llamado retos-javascript.

- [ ] Días 29 al 33 (Sintaxis básica): Un día para declarar variables y tipos de datos, otro día para operaciones matemáticas, otro para condicionales (if/else), otro para bucles (for/while), y otro para funciones. Cada día subes un archivo .js con 3 ejemplos sencillos.

- [ ] Días 34 al 38 (Estructuras de datos): Prácticas diarias manipulando Arrays (listas) y Objetos. Sube ejercicios donde filtres listas, agregues elementos o modifiques datos.

- [ ] Días 39 al 42 (Manipulación del DOM): Aprende a conectar tu JS con el HTML. Sube ejercicios donde, al hacer clic en un botón, cambie el color de fondo de la página o aparezca un texto.

<br> **Semanas 7 y 8:** Proyectos Interactivos (Días 43 a 60)
Ya tienes las herramientas. Ahora harás pequeños proyectos que te lleven de 3 a 4 días cada uno. Recuerda hacer commits por cada funcionalidad que agregues (ej: "Agregado botón de borrar", "Estilos de la app finalizados").

- [ ] Días 43 al 47: Una Calculadora funcional.

- [ ] Días 48 al 52: Una aplicación de Lista de Tareas (To-Do List) donde puedas agregar, marcar como completadas y borrar tareas.

- [ ] Días 53 al 57: Un Juego de adivinar el número (la PC piensa un número del 1 al 100 y tú arriesgas, diciéndote si es más alto o más bajo).

- [ ] Días 58 al 60 (Optimización): Vuelve a tus proyectos anteriores, limpia el código que huela mal, añade comentarios explicativos y actualiza los archivos README.md de cada repositorio explicando qué hiciste.
